문제 원인을 찾았습니다! `API_KEY_ENCRYPTION_KEY` 환경 변수의 값이 유효하지 않아서 발생하는 문제입니다.

**원인 분석:**
- `src/lib/crypto/index.ts` 파일의 코드를 확인해보니, `API_KEY_ENCRYPTION_KEY`는 반드시 **32바이트** 길이의 버퍼로 디코딩되어야 합니다.
- 현재 설정된 키(`q3Z9yN1f...`)는 디코딩 시 **31바이트**가 되어 길이 조건(`!== 32`)을 만족하지 못해 오류가 발생하고 있습니다.

**해결 방안:**
1. **새로운 암호화 키 적용**: 32바이트 길이를 만족하는 올바른 새 키(`pIMNVihNnv6plwke+0sbCrctqUgTjIuo/oLcUlUsVSE=`)를 생성했습니다. 이 키로 `.env`와 `.env.local` 파일을 업데이트하겠습니다.
2. **서버 재시작**: 환경 변수 변경 사항을 적용하기 위해 실행 중인 서버를 재시작합니다.

이 작업을 진행하여 API 키 생성 기능이 정상 작동하도록 수정하겠습니다.
